# ONF Light — Home Assistant 整合

[English](README.md) | [中文](README_zh.md)

![logo](custom_components/onf_light/brand/dark_logo.png)

用於控制 ONF 藍牙植物燈的 Home Assistant 整合（由官方 Android 應用反向工程得來）。透過藍牙低功耗（Nordic UART）協定，提供對受支援 ONF 型號的亮度與色溫（CCT）控制。

本專案應用了官方 ONF 應用中觀察到的相同協定與裝置特定行為，包含：依型號調整的 CCT 範圍、防彈跳與樂觀更新、混合式 BLE 連線策略、裝置端排程，以及自動時鐘同步。

**支援：** ONF BLE 燈（多種型號 — 僅亮度與可調白光型）。僅在 MIST O+ 上有測試。

**警告：** 此整合直接與藍牙裝置通訊，需要 Home Assistant 主機具有可用的藍牙介面與作業系統層級的藍牙權限。

**注意：** 透過 HA 或官方應用手動控制燈光會將裝置切換到手動模式並停用現有排程。要恢復排程運作，請透過選項表單（齒輪圖示）重新推送排程或在官方應用中重新啟用。

---

## 功能

- 亮度與色溫（CCT）控制，色溫以克耳文（Kelvin）為單位
- 型號辨識 — 僅亮度型號可正確處理
- 樂觀更新、防彈跳與回讀確認，減少介面跳動
- 混合式 BLE 連線，支援可設定的閒置斷線時限與逐指令重試
- 裝置無法連線時自動降低輪詢頻率，減少不必要的藍牙流量
- 透過 Home Assistant 藍牙自動發現裝置

### 裝置端排程

最多可在裝置上儲存 5 個自動照明排程，由裝置內部即時時鐘執行。**排程運行期間不需保持藍牙連線。** 只要裝置時鐘已完成同步，排程在斷電後也能繼續執行。

排程可透過整合的選項介面（齒輪圖示）或 HA 服務進行設定。

### 裝置時鐘同步

整合在首次連線時會自動將裝置內部時鐘與 Home Assistant 的時間同步，之後依可設定的間隔定期重複（預設為 24 小時）。同時提供「**同步裝置時鐘**」按鈕實體，可隨時手動同步。

若未進行時鐘同步，裝置端定時器可能會在斷電後發生時間漂移或在錯誤的時間觸發。

### 連線管理

整合的選項介面（齒輪圖示）提供三項連線設定：

| 設定 | 預設值 | 說明 |
| --- | --- | --- |
| 狀態輪詢間隔 | 10 秒 | HA 在背景讀取裝置狀態的頻率。設為 0 可停用輪詢（HA 僅在送出指令後更新狀態）。將此值設定為大於閒置斷線時限，可讓裝置在兩次輪詢之間斷線，有助於與官方 App 共用裝置。 |
| 藍牙閒置斷線 | 30 秒 | 閒置指定秒數後自動斷開藍牙連線。設為 -1 可保持永久連線。 |
| 時鐘同步間隔 | 1440 分鐘 | 自動同步裝置時鐘的頻率。設為 0 可停用。 |

### 診斷實體

以下診斷實體與裝置關聯：

| 實體 | 類型 | 預設 | 說明 |
| --- | --- | --- | --- |
| 藍牙連線狀態 | 二元感測器 | 啟用 | 即時 BLE 連線狀態 — 連線與斷線時立即更新 |
| 已啟用定時器 | 感測器 | 啟用 | 已啟用的排程槽位數量（0–5），屬性中包含各槽位詳細資訊 |
| 韌體版本 | 感測器 | 停用 | 裝置韌體版本字串（例如 `R09A`），首次連線時讀取一次 |
| 訊號強度 | 感測器 | 停用 | 最後一次藍牙廣播的 RSSI（dBm） |
| 色溫範圍 | 感測器 | 停用 | 支援的克耳文範圍（例如 `2700K – 7000K`）或 `Brightness Only` |

預設停用的實體可在實體登錄中個別啟用。

### HA 服務

| 服務 | 說明 |
| --- | --- |
| `onf_light.set_timer` | 設定定時器槽位（槽位 1–5、開啟時間、關閉時間、亮度，CCT 裝置可選填色溫） |
| `onf_light.clear_timer` | 按槽位編號清除，或省略槽位以清除全部 |

- 支援藍牙的 Home Assistant（Linux 搭配 BlueZ、macOS 或支援的藍牙介面）
- Python 3.10+（與 Home Assistant core 需求一致）

## 使用 HACS 安裝

1. 在 Home Assistant 中，前往 HACS → 右上三點選單 → Custom repositories。
2. 新增此 repository 的 URL，分類選 `Integration`。
3. 在 HACS 中搜尋「ONF Light」並點選**安裝**。
4. 重新啟動 Home Assistant。
5. 前往**設定 → 整合**，若附近有 ONF 裝置應會看到發現通知，依提示完成設定。

## 手動安裝

1. 將 `custom_components/onf_light` 資料夾複製到 Home Assistant `config/custom_components` 目錄中。
2. 重新啟動 Home Assistant。
3. 前往**設定 → 整合 → 新增整合**，搜尋 ONF Light。

## 使用說明

### 亮度與色溫

使用標準 HA 燈光控制。亮度採用 0–255 的 HA 比例。色溫以 `color_temp_kelvin` 屬性輸出（克耳文），整合會自動對應至裝置的內部步進精度。

### 定時排程

在**設定 → 整合**中點選整合的齒輪圖示（⚙），可設定最多 5 個定時槽位。每個槽位包含開啟時間、關閉時間、亮度，以及色溫（適用於 CCT 裝置）。排程會推送至裝置並由其內部時鐘執行 — 即使 HA 離線，排程仍可繼續運作。

## 疑難排解

- **裝置變為不可用：** 確認藍牙介面已啟用且在範圍內。裝置無法連線時，整合會自動降低輪詢頻率。如有需要，重新載入整合。
- **更改未生效：** 檢查 Home Assistant 日誌是否有 BLE 通訊錯誤，並確認裝置已開機且未被其他應用獨佔連線。將輪詢間隔設定為大於閒置斷線時限，可讓官方 App 在兩次輪詢之間使用裝置。
- **定時器未在正確時間觸發：** 使用「同步裝置時鐘」按鈕手動重新同步裝置即時時鐘。整合在連線時會自動同步，但長時間斷電後裝置時鐘可能發生漂移。
